Light-Following Robot

Tracks a moving light source with tuned motor control to stay stable under changing ambient lighting.

Case structure: Problem → Constraints → Approach → Outcome → Next iteration

Embedded C Arduino LDR / Photodiodes H-bridge PID control

Problem: Build a small robot that can lock onto and follow a hand-held light source in real time, without oscillating or drifting when the environment changes.

Constraints: Low-cost LDR sensing, noisy ambient light, tight response time, and limited current for motors and logic.

Approach: Arranged an LDR array to give directional hints, normalized readings, and experimented with thresholding before moving to a simple PID-style loop to modulate each wheel’s speed.

Outcome: The robot tracks a flashlight with noticeably smoother turns and fewer stalls, and stays more stable under mixed indoor lighting after tuning gains and adding basic filtering.

Next iteration: Improve behavior under occlusion, add better smoothing or a lightweight filter, and explore small ML models to classify target vs background light.

Gesture Vocalizer Glove

Converts defined hand gestures into spoken phrases, focusing on comfort, latency, and assistive communication.

Case structure: Problem → Constraints → Approach → Outcome → Next iteration

Wearable Flex sensors / IMU Microcontroller Bluetooth / Serial TTS companion

Problem: Enable hands-free or speech-limited users to trigger common phrases using simple, repeatable hand gestures.

Constraints: Comfortable glove fit, variable hand sizes, sensor drift and noise, and a strict end-to-end latency target around [latency] ms.

Approach: Mapped flex sensor bends and IMU orientation to a small vocabulary of gestures, added calibration steps, and filtered readings before sending compact payloads over serial/Bluetooth to a companion app for text-to-speech.

Outcome: The prototype reliably triggers a core phrase set in quick tests, exposing where false positives appear and how calibration flow affects real users.

Next iteration: Move towards on-device gesture classification with small ML models, expand vocabulary, and refine the glove hardware to be lighter and easier to wear for longer sessions.

Projects Website

A fast, accessible portfolio with dark mode, design tokens, and deeper case-study write-ups.

Case structure: Problem → Constraints → Approach → Outcome → What I learned

Next.js / Vite React Semantic HTML CSS variables Dark mode

Problem: Replace scattered links with a single, coherent portfolio that reads like a set of engineering case studies instead of disconnected demos.

Constraints: High performance, accessible color contrast, simple deployment, and a layout that adapts cleanly from mobile to desktop.

Approach: Built a SPA/MPA using React with semantic landmarks, a token-driven color and type system, and support for dark mode via CSS variables and media queries.

Outcome: The site achieves strong performance scores and clearer storytelling, making it easier to see how each project evolved from problem to next iteration.

What I learned: The impact of color contrast, focus management, and consistent IA on how technical work is understood by reviewers and collaborators.
